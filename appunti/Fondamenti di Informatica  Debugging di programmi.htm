Debugging di programmi


∞Probabilmente il primo bug trovato in un programma
Per risolvere un problema di calcolo dobbiamo dapprima pensare a un algoritmo risolutivo. Questo algoritmo andrà poi tradotto in un linguaggio di programmazione come Java per ottenere un programma effettivamente eseguibile da un calcolatore. Scrivere programmi che funzionano perfettamente sin dall'inizio è però quasi impossibile, anche se l'algoritmo che è alla sua base è corretto. Dobbiamo essere quindi in grado di identificare eventuali errori nei nostri programmi e correggerli. Questo compito è universalmente noto come debugging∞, cioè la rimozione dei bug∞ (errori).

Se questo vi sembra un problema da niente, ecco una lista di eventi catastrofici famosi∞ dovuti a bug nascosti nel software e non corretti in tempo dai programmatori!

In questo articolo illustreremo la tecnica più semplice per scovare bug nei nostri programmi: quella delle stampe a video. L'idea è quella di aggiungere in punti opportuni del nostro programma delle operazioni di stampa su schermo che ci permettono di tracciare il funzionamento del programma stesso, ispezionando il contenuto delle sue variabili e identificano la posizione corrente durante l'esecuzione. Forse non avrebbe salvato nel 1996 l'Ariane 5 dall'autodistruzione∞ entro 40 secondi dal decollo per un problema nel software del sistema di riferimento inerziale, ma per i nostri scopi è imbattibile.

Per illustrare il procedimento di debugging mediante stampe, useremo come esempio l'esercizio dato al compito A dell'appello del 12 Gennaio 2007∞.

Problema di calcolo

Scrivere un metodo espansa che, data una matrice v di oggetti String (con un numero arbitrario di righe M e di colonne N), restituisce una nuova matrice con 2*M righe e 2*N colonne ottenuta dalla matrice originale "espandendo" ogni elemento in una matrice 2*2 di elementi uguali.

Ad esempio, data in ingresso la matrice:

A B C
D E F

la matrice prodotta deve essere:

A A B B C C
A A B B C C
D D E E F F
D D E E F F

Scrivere la soluzione in modo da poter compilare il seguente programma di prova senza doverlo modificare:

public class ProvaEserc2 {
    public static void main(String[] args){
        String[][] u = { { "a00", "a01" },
                         { "a10", "a11"}};
        String[][] r = Eserc2.espansa(u);
        System.out.println("TEST 1:");
        stampa("Matrice originale:", u);
        String[][] c = { { "a00", "a00", "a01","a01" },
                         { "a00", "a00", "a01","a01" },
                         { "a10", "a10", "a11","a11"},
                         { "a10", "a10", "a11","a11"}};
        stampa("Matrice espansa calcolata:", r);
        stampa("Matrice espansa corretta:", c);

        String[][] u2 = { { "a00", "a01" }};
        String[][] r2 = Eserc2.espansa(u2);
        System.out.println("TEST 2:");
        stampa("Matrice originale:", u2);
        String[][] c2 = { { "a00", "a00", "a01","a01" },
                          { "a00", "a00", "a01","a01" }};
        stampa("Matrice  espansa calcolata:", r2);
        stampa("Matrice  espansa corretta:", c2);
    }

    private static void stampa(String s, String[][] v){
        System.out.println(s);
        for (int i=0; i<v.length; i++) {
            for (int j=0; j<v[0].length; j++) System.out.print(v[i][j]+" ");
            System.out.println();
        }
    }
}


Prima soluzione (quasi corretta)

Un possibile algoritmo di calcolo consiste nell'allocare dapprima spazio per la matrice di uscita, di cui conosciamo le dimensioni essendo esse il doppio di quelle della matrice di ingresso. Per riempire le celle cella matrice di uscita, scorriamo poi ogni cella (i,j) della matrice di ingresso, e replichiamo il valore della cella nelle quattro celle della matrice di uscita con righe comprese tra 2i e 2i+1 e colonne comprese tra 2j e 2j+1.

Proviamo a implementare questo algoritmo in Java come segue (si veda il post di leopard∞ a riguardo):

public class Eserc2{
    public static String[][] espansa(String[][] v){   
        String[][] s = new String[2*v.length][2*v[0].length];
        int p=0;
        int t=0;
        for (int i=0; i<v.length; i++) {
            for (int j=0; j<v[0].length; j++) {
                for (int k=t; k<t+2; k++) {
                    for (int h=p; h<p+2; h++) {
                        s[k][h]=v[i][j];
                    }
                }
                p=p+2;
            }
            t=t+2;
        }
        return s;   
    }
}


Nel codice sopra, i due cicli for più esterni scandiscono le celle v[i][j] della matrice di ingresso v, mentre i due cicli for più interni servono per replicare il valore della cella v[i][j] nel corrispondente sottoquadrato di lato 2 della matrice di uscita s che parte da s[t][p] e si estende a s[t+1][p+1]. Notiamo che gli indici p e t della matrice di uscita partono da zero e vengono incrementati di 2 ogni volta che i corrispondenti indici i e j della matrice di ingresso vengono incrementati di 1. L'idea è che in questo modo t sia sempre il doppio di i e p sia sempre il doppio di j

Proviamo a compilare ed eseguire il programma di prova dato:

> javac ProvaEserc2.java
> java ProvaEserc2
Exception in thread "main" java.lang.ArrayIndexOutOfBoundsException: 4
        at Eserc2.espansa(Eserc2.java:9)
        at ProvaEserc2.main(ProvaEserc2.java:5)


Il programma viene compilato senza errori, ma ci segnala un errore di esecuzione di tipo ArrayIndexOutOfBoundsException alla riga 9 del file Eserc2.java. La riga in questione è:

s[k][h]=v[i][j];

Il problema sembra essere che almeno uno degli indici utilizzati per accedere a s[k][h] oppure a v[i][j] non è valido, cioè sfora i limiti della matrice.

Debugging mediante stampe

Per scovare il problema, proviamo ad aggiungere una stampa dei valori degli indici i, j, k e h usati dall'assegnazione s[k][h]=v[i][j]; subito prima di eseguirla, e stampiamo inoltre all'inizio le dimensioni delle matrici di ingresso e di uscita:

public class Eserc2{
    public static String[][] espansa(String[][] v){   
        String[][] s = new String[2*v.length][2*v[0].length];
        System.out.println("[debug] v ha dimensioni "+v.length+"x"+v[0].length);
        System.out.println("[debug] s ha dimensioni "+s.length+"x"+s[0].length);
        int p=0;
        int t=0;
        for (int i=0; i<v.length; i++) {
            for (int j=0; j<v[0].length; j++) {
                for (int k=t; k<t+2; k++) {
                    for (int h=p; h<p+2; h++) {
                        System.out.println("[debug] s["+k+"]["+h+"]=v["+i+"]["+j+"]");
                        s[k][h]=v[i][j];
                    }
                }
                p=p+2;
            }
            t=t+2;
        }
        return s;   
    }
}

Le stampe rivelano che la cella (0,0) della matrice di ingresso v viene correttamente copiata nelle quattro celle (0,0), (0,1), (1,0), (1,1) della matrice di uscita s. Lo stesso vale per la cella (0,1) di v, che viene correttamente copiata nelle quattro celle (0,2), (0,3), (1,2), (1,3) di s. Purtroppo però passando alla prima cella della seconda riga di v, cioè (1,0), l'assegnazione tenta di scrivere sulla cella (2,4) di s, che non esiste (gli indici di colonna di s vanno da 0 a 3, e quindi il fatto che h vale 4 genera un errore di tipo ArrayIndexOutOfBoundsException).

Perché h arriva a valere 4? Per capirlo, ricordiamo che h va da p a p+1, quindi dovremmo studiare il valore di p. Aggiungiamo un'altra stampa che ci mostra i valori di p e t corrispondenti a una data coppia di i e j:

public class Eserc2{
    public static String[][] espansa(String[][] v){   
        String[][] s = new String[2*v.length][2*v[0].length];
        System.out.println("[debug] v ha dimensioni "+v.length+"x"+v[0].length);
        System.out.println("[debug] s ha dimensioni "+s.length+"x"+s[0].length);
        int p=0;
        int t=0;
        for (int i=0; i<v.length; i++) {
            for (int j=0; j<v[0].length; j++) {
                System.out.println("[debug] i="+i+" | j="+j+" | t="+t+" | p="+p);
                for (int k=t; k<t+2; k++) {
                    for (int h=p; h<p+2; h++) {
                        System.out.println("[debug] s["+k+"]["+h+"]=v["+i+"]["+j+"]");
                        s[k][h]=v[i][j];
                    }
                }
                p=p+2;
            }
            t=t+2;
        }
        return s;   
    }
}

Risulta chiaro che viene violata la proprietà secondo cui p deve essere il doppio di j. Il punto è che p continua a crescere sempre di 2, mentre passando alla riga successiva di v, p dovrebbe azzerarsi come accade per j!

L'errore è che l'istruzione int p=0; dovrebbe essere eseguita ogni volta che si inizia a scandire una nuova riga di v, quindi int p=0; dovrebbe essere portata all'inizio del corpo del primo for, prima di entrare nel secondo for.

Seconda soluzione (corretta)

Modifichiamo il programma come appena detto:

Ora tutto sembra essere corretto, p si riazzera come dovrebbe! Possiamo quindi rimuovere le stampe di debugging e il nostro programma finale è pronto:

public class Eserc2{
    public static String[][] espansa(String[][] v){   
        String[][] s = new String[2*v.length][2*v[0].length];
        int t=0;
        for (int i=0; i<v.length; i++) {
            int p=0;
            for (int j=0; j<v[0].length; j++) {
                for (int k=t; k<t+2; k++) {
                    for (int h=p; h<p+2; h++) {
                        s[k][h]=v[i][j];
                    }
                }
                p=p+2;
            }
            t=t+2;
        }
        return s;   
    }
}
